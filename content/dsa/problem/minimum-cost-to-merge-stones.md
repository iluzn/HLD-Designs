<h1>Minimum Cost to Merge Stones</h1>
<span class="lc-diff hard">Hard</span>
<details class="lc-topics"><summary>Topics</summary><div class="lc-tags"><span class="lc-tag">Array</span><span class="lc-tag">Dynamic Programming</span><span class="lc-tag">Prefix Sum</span></div></details>
<p>You have <code>nums.length</code> piles of stones arranged in a row; <code>nums[i]</code> is the number of stones in pile <code>i</code>. In one move you merge exactly <code>k</code> consecutive piles into a single pile, and the cost of that move equals the total number of stones in those <code>k</code> piles. Return the minimum total cost to merge all piles into one pile. If it is impossible, return <code>-1</code>.</p>
<h2>Example 1</h2><pre>Input:  stones = [3,2,4,1], k = 2
Output: 20</pre>
<h2>Example 2</h2><pre>Input:  stones = [3,2,4,1], k = 3
Output: -1</pre>
<h2>Example 3</h2><pre>Input:  stones = [3,5,1,2,6], k = 3
Output: 25</pre>
<h2>Constraints</h2><ul><li>1 &lt;= nums.length &lt;= 30</li><li>2 &lt;= k &lt;= 30</li><li>1 &lt;= nums[i] &lt;= 100</li></ul>
